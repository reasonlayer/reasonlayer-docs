Adding a credential is a two-step process: first add it in the Credentials tab, then assign it to the agent that will use it.

As an example, we're going to add an `OPENROUTER_API_KEY` to the vault to show you how it works.

## Steps

1. **Get your API Key from the provider**: in this case I went to Openrouter and generated a key.
2. **Create a credential in Reasonlayer**: In the credentials tab, create a new credential with a friendly name, the environment variable name gets set later so this is just for internal purposes:
   - **Name:** `Dev Openrouter API key`
   - **Value:** `sk-...`
   - **Allowed domains:** `*.openrouter.ai`

We allow wildcards in the allowed domains field (e.g., `*.openrouter.ai` matches all subdomains of openrouter.ai).

3. Go to your agent, select the credential you want to assign. You have to choose an environment variable name you'd like to use for this credential.

### You're done!

To use the credential, you have to replace or add the credential in your OpenClaw's config files and everywhere these credentials are used.

You should now do a OpenClaw gateway restart.

Please see [Using credentials in OpenClaw](/using-reasonlayer/credentials-in-openclaw).
